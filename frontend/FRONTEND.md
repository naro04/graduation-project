# واجهة المشروع الأمامية (Frontend)

## نظرة عامة

الواجهة مبنية بـ **React 18** و **Vite 5**، وترتبط بالباكند عبر **REST API**. جميع البيانات المعروضة (أقسام، مناصب، أدوار، مواقع، موظفين، تقارير) تأتي من الـ API وليست قوائم ثابتة.

## هيكل المشروع

```
src/
├── components/     # مكونات React (صفحات، مودالات، عناصر مشتركة)
├── services/       # طبقة الاتصال بالـ API (auth, employees, departments, locations, ...)
├── utils/          # دوال مساعدة (مثل تحويل روابط الصور، تصدير التقارير)
├── config/         # إعدادات (مثل قائمة القائمة الجانبية)
├── App.jsx         # نقطة الدخول للتطبيق والمسارات
└── main.jsx        # ربط التطبيق بالـ DOM
```

## الاتصال بالـ API

- **العنوان الأساسي:** يُعرّف في `services/apiClient.js` (Axios) ويُستخدم في كل طلبات الـ API.
- **المصادقة:** توكن JWT يُحفظ في `localStorage` تحت المفتاح `authToken` ويُرسل في ترويسة `Authorization`.
- **الخدمات:** كل وحدة في `services/` تتوافق مع جزء من الباكند (مثلاً `employees.js`، `departments.js`، `locations.js`، `rbac.js`، `profile.js`).

## مصدر البيانات

- **الأقسام والمناصب والأدوار:** من `getDepartments()`، `getPositions()`، `getRoles()` وتُستخدم في الفلاتر والنماذج.
- **المواقع وأنواع المواقع:** من `getLocations()`، `getLocationTypes()`.
- **الموظفون والتقارير:** من `getEmployees()`، `getHRReports()` وغيرها.
- **الصور (الأفاتار):** روابط كاملة من الباكند (مثل Cloudinary) أو مسارات نسبية تُحوَّل إلى روابط مطلقة عبر `utils/avatarUrl.js`.

## التشغيل والبناء والاختبار

```bash
# التطوير (سيرفر محلي)
npm run dev

# بناء للإنتاج
npm run build

# تشغيل الاختبارات
npm run test
```

## الاختبارات

يوجد اختبار وحدة لـ `utils/avatarUrl.js` (دالة تحويل روابط الصور) باستخدام **Vitest**. تشغيل الاختبارات: `npm run test`.
